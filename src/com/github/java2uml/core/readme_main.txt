При вызове метода main() надо передать параметры (регистр значения не имеет):
1й параметр (обязательный) - тип исходных файлов. Допустимые варианты: "class", "java". Можно сокращать до 3 символов.
2й параметр (обязательный) - путь к исходным файлам (если содержит пробелы, ОБЯЗАТЕЛЬНО заключать в двойные кавычки).
        Можно просто указать путь, можно добавить имя параметра "src" или "source_path" (можно сократить до 3 символов).
        Пример: "D:\Some Project\src", src="D:\Some Project\src", sour="D:\Some Project\src" и т.п.

Необязательные параметры (могут располагаться в любой последовательности после первых двух обязательных):
output_file - задает имя (не путь) выходного файла с кодом plantUML (текстовый). Имя параметра можно сократить
        до 3 символов. Если параметр не указан, будет задано относительное имя "classes.plantuml".
        Пример: out=file.txt, output="d:\Some Project\plantUML\code.txt" и т.п.
header - задает заголовок (если заголовок содержит пробелы, то ОБЯЗАТЕЛЬНО заключать в двойные кавычки).
        Имя параметра можно сократить до 3 символов.
        Пример: head=Title, header="Заголовок диаграммы" и т.п.
classes_diagram (по умолчанию) | sequence_diagram - задают тип формируемой диаграммы. Не могут быть указаны совместно.
        Можно сократить до 3 символов.
vertical (по умолчанию) | horizontal - задают направление диаграммы. Не могут быть указаны совместно.
        Можно сократить до 3 символов.
nocomposition | noaggregation | noassociation | nolollipop - отключают вывод соответствующих связей. Могут быть указаны
        в любых сочетаниях. Можно сократить до 5 символов.

В методе main() все параметры будут скомпонованы в объект класса Options, который будет передан в обрабатывающий метод.
